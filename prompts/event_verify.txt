operator's calibration loop -- be HONEST, not generous. A prediction is "hit"
ONLY when the post-window news contains a CONCRETE confirmation of the
predicted outcome. "Stock went up" is not a hit if the predicted event was
"earnings beat" -- you need an actual earnings release that shows the beat.

Verdict trichotomy + an extra "partial":
- `hit`        : evidence concretely confirms the predicted outcome.
- `miss`       : evidence concretely refutes it (opposite direction or it didn't happen).
- `partial`    : event happened but the outcome was mixed (e.g. revenue beat, EPS miss).
- `pending`    : the event hasn't happened yet OR no decisive evidence either way.

Rules:
1. Only cite evidence that appears in the provided news block. No fabrication.
2. "Stock price went up/down" is ONLY admissible for technical/sentiment events,
   NOT for fundamental events (earnings, guidance, contract win, regulatory).
3. If the predicted_outcome is multi-part, all parts must hit for "hit"; if
   some hit and others miss, mark "partial".
4. If the window has not yet started or has barely begun, default to "pending".
5. evidence_text should be 1-2 short sentences quoting/paraphrasing the
   decisive headline or filing line. Never empty for hit/miss/partial.

Output strict JSON. No prose, no markdown fences, no <think> tags.
