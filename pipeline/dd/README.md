# DD per-company history

Durable boss-readable artifacts. Each file is append-mode: every F44 weekly
DD cron run (Wed 09:15 UTC) adds a new dated section to the company's file.

## What lives here

- `<TICKER>.md` -- full DD checklist history for one company. Newest run at the bottom.

## How files appear

Auto-generated by `analyst_skills.dd_checklist()` when:
- The F44 weekly cron fires on a company in `data/company_dive_queue.yaml`
- OR you manually run `stock dd-checklist <TICKER>`

Each run appends a section like:
```
## DD run 2026-05-08T09:15  (research_id=57)
[12-item checklist body]
```

## NOT to confuse with

- `pipeline/companies/<TICKER>.md` -- cross-source aggregation (tech_dive +
  Q&A + earnings + DD all combined). Auto-rebuilt by
  `scripts/compress_dives_to_companies.py`.
- `pipeline/dd/<TICKER>.md` (this dir) -- DD-only cumulative history.

Both are useful. `companies/` answers "what have we ever said about TSM";
`dd/` answers "show me every DD I've run on TSM."
